* SPEC: TreasureEarn (Get-Paid-To Platform) – Restart Requirements
Goal
Build a production-ready GPT website with:
* Ledger-based balances (available/pending/locked)

* Offer click tracking + postback system (auto credit)

* Withdraw workflow (user request → admin review → payout)

* Security + anti-fraud baseline

* Modern black + gold UI

P0 (Must-have để chạy được + apply offerwalls)
1) Auth + Account
Pages
   * /register, /login, /logout

   * /verify (email verification)

   * /profile (basic info)

Backend
      * Secure session via httpOnly cookies (JWT or session)

      * Email verification required to withdraw / cashout

      * Basic user model: email, username, createdAt, status, role, referralCode (optional)

UX
         * Block withdraw if not verified → show popup/toast “Verify email to withdraw”.

________________


2) Balance / Ledger System (Core)
Balances
            * available_balance

            * pending_balance

            * locked_balance (hold)

            * lifetime_earned, lifetime_withdrawn

Ledger
               * Every balance change must create an immutable ledger record.

               * Must support idempotency key (prevent double credit).

Ledger event types
                  * earn_pending

                  * earn_approved (move pending → available)

                  * earn_denied/chargeback (revert/clawback)

                  * withdraw_request

                  * withdraw_approve

                  * withdraw_deny

                  * hold_add / hold_release

                  * admin_adjust

________________


3) Offer System (Offer list + Offer detail + click tracking)
Offer list
                     * Pull offers from providers (server-side). Cache/revalidate to avoid spam.

                     * Search + filters (yours) with URL query sync.

                     * Skeleton loading.

Offer detail
                        * Description/terms + steps/tasks if provider supports.

                        * “Start Offer” button triggers click session.

Click session tracking (MUST)
                           * When user clicks start:

                              * create session record with: userId, provider, offerId, clickId/sessionId, IP/device fingerprint (basic), createdAt

                              * redirect to provider tracking link including clickId/subid

                                 * Without click tracking, postback fraud detection will be weak.

________________


4) Postback / Callback System (Critical)
Requirement: provider must be able to send events back
Endpoints
                                    * One unified endpoint or per-provider endpoints.

                                    * Must verify provider secret (token/HMAC/IP allowlist depending on provider).

Must store
                                       * raw payload + headers

                                       * provider name

                                       * eventId (unique)

                                       * clickId/subId

                                       * offerId

                                       * status (pending/approved/denied/chargeback)

                                       * payout amount

                                       * timestamp

Must enforce
                                          * Duplicate prevention: unique index on (provider + eventId)

                                          * Status validation: do NOT accept approved without a known click session (or mark suspicious & hold)

                                          * Rate limiting on postback routes

                                          * Logging + audit trail

Async processing
                                             * Postback endpoint should enqueue job and respond fast.

                                             * Worker updates ledger + balances.

Balance rules
                                                * pending → add to pending_balance

                                                * approved → move pending → available

                                                * denied/chargeback → clawback/revert with ledger record

________________

                                                                                                                                                                                                                                                         * HMAC hoặc token so sánh với secret lưu trong server

                                                                                                                                                                                                                                                         * Nếu sai → reject

B. Validate dữ liệu
                                                                                                                                                                                                                                                            * userId phải tồn tại

                                                                                                                                                                                                                                                            * offerId tồn tại

                                                                                                                                                                                                                                                            * amount > 0

                                                                                                                                                                                                                                                            * status hợp lệ

C. Deduplicate
                                                                                                                                                                                                                                                               * Mỗi provider gửi transaction_id

                                                                                                                                                                                                                                                               * Tạo unique index: (provider + transaction_id)

                                                                                                                                                                                                                                                               * Nếu tồn tại rồi → ignore

D. Log raw payload
                                                                                                                                                                                                                                                                  * Lưu toàn bộ body + headers vào postback_logs

4️⃣ OFFER HOLD POLICY (CỤ THỂ)
Hệ thống phải hỗ trợ HOLD theo rule:
4.1 Hold mặc định
Ví dụ:
                                                                                                                                                                                                                                                                     * Game offers → hold 7 ngày

                                                                                                                                                                                                                                                                     * Crypto offers → hold 14 ngày

                                                                                                                                                                                                                                                                     * Casino offers → hold 30 ngày

                                                                                                                                                                                                                                                                     * Survey → hold 0–3 ngày
0-10$ hold 1 ngày, 11-25$ hold 1 tuần, 26-50 2 tuần, 51-100 4 tuần, $100-100,000 6 tuần
4.2 Hold theo risk level
Nếu user risk cao:
                                                                                                                                                                                                                                                                        * Tăng hold thêm X ngày (2 tuần)

                                                                                                                                                                                                                                                                        * Hoặc chuyển sang manual review

4.3 Flow xử lý
Nếu status = pending:
→ cộng vào pending_balance
Nếu status = approved:
→ tạo ledger:
                                                                                                                                                                                                                                                                           * credit_pending

→ set unlockAt = now + holdDays

Cron job mỗi giờ:
                                                                                                                                                                                                                                                                              * Check completion đã hết hold

                                                                                                                                                                                                                                                                              * Chuyển:

pending_balance → available_balance

                                                                                                                                                                                                                                                                              * Tạo ledger record: release_to_available

Nếu status = chargeback:
→ tạo reversal ledger
→ trừ available hoặc pending tùy trạng thái

                                                                                                                                                                                                                                                                                 * status

                                                                                                                                                                                                                                                                                 * referenceId

                                                                                                                                                                                                                                                                                 * provider

                                                                                                                                                                                                                                                                                 * createdAt

Loại ledger:
                                                                                                                                                                                                                                                                                    * offer_pending

                                                                                                                                                                                                                                                                                    * offer_release

                                                                                                                                                                                                                                                                                    * offer_reversal

                                                                                                                                                                                                                                                                                    * withdraw_request

6️⃣ WITHDRAW SYSTEM (LOGIC CỤ THỂ)
6.1 Điều kiện rút
User chỉ được rút nếu:
                                                                                                                                                                                                                                                                                       * emailVerified = true

                                                                                                                                                                                                                                                                                       * available_balance >= amount

                                                                                                                                                                                                                                                                                       * không bị banned

                                                                                                                                                                                                                                                                                       * riskScore < threshold (hoặc manual review)

6.2 Flow
User request:
                                                                                                                                                                                                                                                                                          * tạo withdraw record status = pending

                                                                                                                                                                                                                                                                                          * trừ available_balance bằng ledger debit_withdraw

Admin approve:
                                                                                                                                                                                                                                                                                             * update status = approved

                                                                                                                                                                                                                                                                                             * lưu payout reference (tx hash / PayPal ID)

                                                                                                                                                                                                                                                                                             * status = paid

Admin deny:
                                                                                                                                                                                                                                                                                                * update status = denied

                                                                                                                                                                                                                                                                                                * hoàn tiền lại available_balance

6.3 Anti abuse
                                                                                                                                                                                                                                                                                                   * Cooldown giữa các lần withdraw (ví dụ 24h)

                                                                                                                                                                                                                                                                                                   * Giới hạn rút mỗi ngày (5)

                                                                                                                                                                                                                                                                                                   * Nếu withdraw đầu tiên → auto hold 24–72h

                                                                                                                                                                                                                                                                                                   * Nếu risk cao → manual review bắt buộc

7️⃣ RISK SYSTEM (LOGIC CỤ THỂ)
Risk score 0–100 dựa trên:
                                                                                                                                                                                                                                                                                                      * VPN detected (+20)

                                                                                                                                                                                                                                                                                                      * IP thay đổi > 3 lần/ngày (+15)

                                                                                                                                                                                                                                                                                                      * Chargeback history (+30)

                                                                                                                                                                                                                                                                                                      * Hoàn thành quá nhanh (+10)

                                                                                                                                                                                                                                                                                                      * Nhiều account chung IP (+25)

Nếu risk > 60:
                                                                                                                                                                                                                                                                                                         * auto hold earnings

                                                                                                                                                                                                                                                                                                         * block withdraw

Nếu risk > 80:
                                                                                                                                                                                                                                                                                                            * require KYC hoặc ban
